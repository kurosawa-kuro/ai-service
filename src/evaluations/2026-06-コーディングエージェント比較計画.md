# 2026-06 コーディングエージェント比較計画

## タスク

同じ小〜中規模 repo 作業を、複数のコーディングエージェントで比較する。

## 入力

- 既存 repo
- 要件 docs
- 変更対象ファイル
- テストコマンド
- `AGENTS.md` / `CLAUDE.md`

## 使用サービス / 技術

- Codex / Codex CLI
- Claude Code
- Aider
- Cline
- OpenCode
- GitHub Copilot
- Cursor

## 比較タスク

| タスク | 見ること |
|---|---|
| Markdown台帳の項目追加 | docs構造の理解、余計な変更の少なさ |
| 小さなCLI修正 | 既存コード追従、テスト追加 |
| テスト失敗の修正 | 原因分析、最小修正 |
| README / routing-rules更新 | 仕様と実装の整合 |
| リファクタ候補整理 | 過剰設計を避けられるか |

## 評価軸

| 軸 | A | B | C |
|---|---|---|---|
| 差分品質 | 小さく意図通り | 手直しすれば使える | 余計な変更が多い |
| 既存理解 | docs / code / Makefile を踏まえる | 一部抜ける | 文脈を外す |
| 検証 | テストまで回す | コマンド提案止まり | 検証なし |
| 安全性 | 破壊的操作を避ける | 注意すれば使える | 危険 |
| 再現性 | 手順が残る | 断片的 | 残らない |

## 再利用判断

- repo実装の第一候補は Codex / Claude Code で検証する。

- Aider / Cline / OpenCode は、代替候補として同一タスクで比較する。

## 結論

この2週間で、競争軸が「チャット性能」から「長時間稼働する実務エージェント」と「価格性能比」へ移った。最大の変化はモデルの知能向上ではなく、**モデル階層・推論強度・実行環境・Prompt 管理を分離できるようになった**こと。

特に重要な5件:

1. OpenAI: GPT-5.6（3階層）と ChatGPT Work
2. Anthropic: Claude Sonnet 5（デフォルト化＋プロモ価格）
3. xAI: Grok 4.5（出力 $6 の原価破壊）
4. Google: Gemini Spark のデスクトップエージェント化
5. Mistral: Prompt / Skill の System of Record 化

「高性能モデルを能力の中心に置かず、通常処理は安価なモデルに任せ、高影響時だけ上位を呼ぶ」という既存の設計方針は、各社の製品構造そのものになり始めており、変更不要。

構造の要点: モデル名の乱立を **Sol / Terra / Luna（能力・原価階層）× High / Max / Ultra（推論計算量）** の二軸に整理。GPT-5.6 Sol はベンチマーク首位狙いではなく、長時間ワークフロー・ツール呼び出し削減（GPT-5.5 比で工程数 −25%、ツール呼び出し −35〜48% の事例）・エージェントの迷走耐性を強化。

役割分離: Codex＝リポジトリ・コード変更 ／ ChatGPT Work＝複数アプリ・文書・業務成果物 ／ 通常 ChatGPT＝相談・裁定。

### xAI

- 7/8 **Grok 4.5** 公開。コーディング・エージェント・工学科学・ナレッジワーク向けフラッグシップ。Cursor と共同で訓練・評価。

## 有力シナリオ

| シナリオ | 確度 | 要旨 |
|---|---|---|
| A. OpenAI が「モデル＋実行環境」で主導権 | 高 | GPT-5.6 単体より ChatGPT / Work / Codex / PTC / Multi-agent / GPT-Live の統合が強い。「賢いモデル」競争から「仕事を完了する OS」競争へ |
| B. Anthropic は上位裁定とコーディングを握る | 高 | Sonnet 5 / Opus 4.8 / Fable 5 / Mythos 5 の明確な階層。弱点は価格と規制リスク |
| C. Grok / GLM が実装原価を破壊 | 中〜高 | 出力 $6 とオープンウェイトが「全処理を Sonnet / GPT で回す」経済合理性を崩す。上位が考え安価モデルが大量実行する構造が標準化 |
| D. Prompt 工学が「設定ファイル」から「運用システム」へ | 非常に高 | Mistral の System of Record 化が先行指標。Git 管理だけでなく、どこで・誰に・どのモデルで適用され結果がどうだったかの追跡が必要になる |
| E. 「フロンティア＝消費者プラン外」構造の崩壊 | — | GPT-5.6 一般公開と Fable 5 再展開で反転。消費者ティア比較表は再び有効。職場エージェントは ChatGPT Work vs Claude Cowork の二強化 |

## 破綻条件

- GPT-5.6 の実運用で Ultra / Multi-agent の原価と待ち時間が重すぎる
- Sonnet 5 が Fable との差を縮めすぎ、上位モデルを呼ぶ意味が薄くなる（逆に Pro 降格の論拠は強まる）
- Grok 4.5 が長時間タスクで不安定、または API 運用品質が低い
- ChatGPT Work / Gemini Spark が権限・安全確認だらけで実務速度を落とす
- **AI ハーネス管理の肥大化**: Mistral の Prompt 管理思想は正しいが、過剰な hook・承認・scope 警告・証拠台帳など「owner を縛る官僚機構」へ戻さない。管理対象は AI の挙動であり、owner の判断ではない（CLAUDE.md の方針と同じ）

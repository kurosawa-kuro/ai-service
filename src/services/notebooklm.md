## ハーネス適性

- ワークフロー組み込み: 中
- 接続方式: Web UI中心想定
- 制約: 入力資料の扱い、共有範囲、出力の検証が必要

## 強み

- 手元資料を起点に調査できる。
- 長い資料の要約や論点整理に向く。
- 講座資料・営業資料・調査メモの読み込みに使える可能性がある。

## 弱み

- API / CLI 化しにくい可能性がある。
- 外部最新情報調査は別ツールと分ける。
- アップロード可能な資料の機密性確認が必要。

## 得意用途

- PDF / Docs の要約。
- 資料横断のQ&A。
- 講座素材の構造化。

## 苦手用途

- Web最新情報の一次調査。
- コード実装。
- 機密資料の無確認投入。

## 黒澤用途

- AI講座資料の整理。
- 顧問 / PM 資料の読み込み。
- トレード材料の手元資料整理。

## ハーネス適性

- ワークフロー組み込み: 中〜高
- 接続方式: IDE / GitHub
- 制約: 補完中心のため、repo 全体を任せる作業は Codex / Claude Code と分ける

## 強み

- エディタ内の小さな補完。
- 反復的なコード記述。
- GitHub 連携。

## 弱み

- 大規模設計や広範囲リファクタでは主導役にしにくい。
- 補完の便利さが評価ログに残りにくい。

## 得意用途

- IDE内補完。
- テストの雛形。
- 小さな関数や型定義の補助。

## 苦手用途

- 複数ファイルをまたぐ責務整理。
- docs / Makefile / テストまで含む一連の作業。

## 黒澤用途

- 日常実装の補助。
- Codex / Claude Code を使うほどではない軽い補完。

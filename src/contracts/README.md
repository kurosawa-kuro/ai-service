# 契約管理

黒澤個人の AI サービス契約の台帳。

## 絶対原則

> **フロンティアのハイプランを複数契約しない。**

これは理想形の選択に依存しない、契約判断の不変条件。第一理想・第二理想のどちらを採っていても常に効く。

具体的には、Claude Max と ChatGPT Pro のような**最高峰ティアの同時保持を禁止**する。
高額固定費の二重化そのものが禁止対象であり、「両方使う場面がある」は理由にならない
（片側は標準プランか API 従量で代替する）。

## 🔒 削除禁止セクション

[AIフロンティア・バーベル戦略.md](<AIフロンティア・バーベル戦略.md>) の
**「フロンティア最安到達プラン」** セクションは恒久管理対象。削除・要約・統合しない。

「各プロバイダのフロンティア最上位モデルに到達できる最安プランはどれか」が、
第一理想と第二理想のどちらを採るかを決める唯一の閾値であり、契約判断の分岐データそのものであるため。
**表に未確認が残っている間は契約変更を実行しない。**

## 理想は2段構え

```text
絶対原則: フロンティアのハイプランを複数契約しない
  ├─ 第一理想: 将来方針（コスト最適・ミドル2枠）
  │    ↓ 動きが速く安定維持が難しい場合
  └─ 第二理想: バーベル戦略（変化耐性・高額1＋標準1＋API）
       ↓
     現状: 現在の契約
```

第一理想はコスト最適だが、**AI の動きが速く、この形を安定させ続けるのが難しい**。
そのため、第一理想が維持できないときの構えとして第二理想（バーベル戦略）を持つ。

2つは競合する案ではなく、優先順位のついた目標形。どちらを採るかで迷ったら第一理想を試み、
安定しないと判断した時点でバーベル戦略へ寄せる。
どちらに寄せる場合も、絶対原則だけは崩さない。

現在の契約は、理想2つとは疎結合で管理する（現契約の変更で理想側を書き換えない。逆も同じ）。

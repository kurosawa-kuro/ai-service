# 使い分けルール

用途から AI サービス / 技術を選ぶための使い分け判断の正本。

## 基本方針

- 能力カテゴリ、用途、ハーネス適性、評価ログを根拠に選ぶ。
- 印象評価だけでルーティングしない。
- 機密に近い内容はローカル実行または契約上許可された環境に限定する。
- 「契約を増やす / 減らす」ではなく、増え続ける AI を比較可能な能力資産として扱う。
- 価格だけで判断しない。API / CLI / MCP / AGENTS.md / ローカル可否 / ログの残しやすさを必ず見る。

## モデル階層の使い分け仮説（2026-07-10 追加）

> 出典: [docs/tasks/active/最新情報.md](../../docs/tasks/active/最新情報.md)（2026-06-26〜07-10 動向）。**二次情報ベースの仮説であり、評価ログでの実測前。** 各社が「モデル階層 × 推論強度 × 実行環境」を分離し始めたため、サービス単位のルールに加えてモデル階層の目安を持つ。

| 役割 | 候補（未検証含む） |
|---|---|
| 日常相談・仕様案・リファクタ案 | Claude Sonnet 5 / GPT-5.6 Terra |
| 大量調査・候補生成・横断実装 | Grok 4.5 / GLM-5.2 / DeepSeek |
| 高難度の新機能設計 | GPT-5.6 Sol（High / Max） |
| OS 全体・巨大設計の再編 | GPT-5.6 Sol Ultra / Claude Fable 5 |
| 最終審査・反証 | GPT-5.6 Sol Pro / Claude Fable 5 / Claude Opus 4.8 |
| ローカル／業務アプリ操作 | ChatGPT Work / Gemini Spark |
| 音声インターフェース | GPT-Live |

- 原則: 高性能モデルを能力の中心に置かず、通常処理は安価なモデルに任せ、失敗時・高影響時・検証不能時だけ上位モデルを呼ぶ。
- プロバイダーリスク（政府規制・提供停止）を評価に含める。単一フロンティアモデルへの固定はしない（Fable 5 一時停止の教訓）。
- 破綻条件: 印象だけで階層を決める。評価ログなしにこの表を正本扱いする。

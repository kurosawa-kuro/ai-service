## 原則整合ステータス：⚠️ 違反中

絶対原則（[README.md](README.md)）は「**フロンティアのハイプランを複数契約しない**」。

現契約は **Claude Max 20x ($200) ＋ ChatGPT Pro ($100)** で、ハイプランを2枚同時に保持しており、この原則に違反している。
第二理想 [AIフロンティア・バーベル戦略.md](<AIフロンティア・バーベル戦略.md>) が核心として名指しで避けるべきとした構成そのもの。

| 時点 | 構成 | 原則適合 |
|---|---|---|
| 現在（〜8/7） | Claude Max 20x $200 ＋ ChatGPT Pro $100 | ❌ ハイプラン2枚 |
| 8/7 に Max 5x へ戻した場合 | Claude Max 5x $100 ＋ ChatGPT Pro $100 | ❌ **依然2枚**（金額が下がるだけで違反は解消しない） |
| 原則を満たす最小変更 | 片側をミドル（$20 級）か API 従量へ落とす | ✅ |

**注意**: 8/7 の判断を「Max 20x → 5x」だけで終わらせると原則違反は続く。
違反解消には、Claude / ChatGPT の**どちらをハイプランとして残すかを選び、他方を降格**する必要がある。
どちらを残すかは [evaluations/](../evaluations/) の実測で決める。

## 役割分担（現契約での使い分け）

- **設計・レビュー・重い実装** → Claude Code（Claude Max 20x、〜8/7。以降は 5x に戻すか判断）。Fable 系の重いタスクは 20x 期間中に消化。
- **repo 実装・docs 同期・CLI エージェント** → Codex（ChatGPT Pro）。
- **IDE 内の補完・軽い修正** → GitHub Copilot Pro。
- **AI IDE 体験の試用** → Cursor Hobby（無料枠）。有料化判断は保留。

## 見直しの観点（コスト最適化）

- **Claude Max 20x ($200・一時)**: Fable 都合で 〜8/7 の1か月間だけアップグレード中。**8/7 の判断**: 20x 期間中の実使用量（Fable 利用の頻度・上限到達履歴）を見て、5x に戻す / 20x 継続 / さらに下げるを決める。
  - **原則制約**: 上の「原則整合ステータス」のとおり、5x へ戻すだけでは ChatGPT Pro とのハイプラン2枚が残り原則違反は解消しない。8/7 は「20x か 5x か」ではなく「**Claude と ChatGPT のどちらをハイプラン枠にするか**」の判断にする。
  - **降格側の判断材料（2026-07-10・未検証）**: Sonnet 5（6/30 公開、Opus 級性能・Claude Code デフォルト化）で「下位プラン枠内で問題天井を超える」仮説は強まった。ただし①週次上限 +50% プロモが **7/13 失効** → 失効後の実効上限を実測してから、② Cowork Web / モバイルは **Max プラン先行**。この2点を確認してから実行。
- **ChatGPT Pro ($100)**: Codex メイン。Codex 中心で quota が余るなら Plus ($20) で足りる可能性。逆に quota 逼迫なら Pro $200。
  - **2026-07-10・未検証**: GPT-5.6（Sol / Terra / Luna）公開、ChatGPT Work は Pro から展開・数日内に Plus 拡大予告。Work が Plus で実用になるなら Plus 降格の論拠、Pro 限定機能（ultra 推論等）が効くなら維持の論拠。
- **重複の点検**: Claude Code / Codex / Copilot で**コーディング支援が三重**。実作業でどれをどれだけ使うかを [evaluations/](../evaluations/) で計測し、被りが大きければ 1つを格下げ/解約する候補。
- **合計 $310/月（≈¥48,050、8/7 以降は $210 想定）** は個人としては小さくない。用途量の実測（`evaluations/`）で「定額サブスク vs BYO-key 従量」の損益分岐を出すと最適化しやすい。目標形は第一理想 [将来方針.md](将来方針.md)（$60〜/月）、それが安定しない場合は第二理想 [AIフロンティア・バーベル戦略.md](<AIフロンティア・バーベル戦略.md>)。

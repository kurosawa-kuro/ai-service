# 評価ログ

実タスクで AI サービス / 技術を使った結果を残す場所。

評価ログは感想ではなく、次回の選択に使う根拠として書く。

## ファイル名

```text
YYYY-MM-<topic>.md
```

例:

- `2026-06-codex-vs-claude-code-refactor.md`
- `2026-06-perplexity-notebooklm-調査.md`
- `2026-06-ollama-local-secret-test.md`

## 書くこと

| 項目 | 内容 |
|---|---|
| タスク | 実施した作業 |
| 入力 | 渡した前提、素材、制約 |
| 使用サービス / 技術 | 比較対象を含める |
| 結果 | 何が返ってきたか、何ができたか |
| 理由 | なぜその評価にしたか |
| 再利用判断 | 次回どの用途で使うか / 使わないか |
| 次回へのメモ | 追加検証、破綻条件、注意点 |

## 評価軸

| 軸 | 見ること |
|---|---|
| 正確性 | 仕様、コード、出典、制約を外していないか |
| 実務適性 | 黒澤用途でそのまま使えるか |
| ハーネス適性 | API / CLI / MCP / ローカル / ログ化に乗るか |
| コスト感 | 無料枠、月額、APIコスト、低コスト代替 |
| 再現性 | 同じ条件で再利用できるか |
| 破綻条件 | どこから危ないか、任せてはいけないか |

## 評価ラベル

| 評価 | 意味 |
|---|---|
| A | 第一候補にできる。明確な勝ち筋がある |
| B | 補助候補。条件付きで使える |
| C | 候補棚に戻す。現時点では優先しない |
| 保留 | 追加検証が必要 |

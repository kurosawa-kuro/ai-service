## ハーネス適性

- ワークフロー組み込み: 中〜高
- 接続方式: IDE
- 制約: IDE固有体験に依存しやすく、作業ログの残し方を決める必要がある

## 強み

- エディタ内で文脈を持った修正を試しやすい。
- UI上で差分を見ながら操作しやすい。
- エージェント型IDEの実験に向く。

## 弱み

- CLI中心のワークフローとは別管理になりやすい。
- 同じファイルを他エージェントと同時編集すると破綻しやすい。

## 得意用途

- UIを見ながらの実装。
- 複数ファイルの軽〜中規模修正。
- IDE内での探索。

## 苦手用途

- 完全にログ化された再現可能な CLI 作業。
- 大規模な仕様整理。

## 黒澤用途

- エージェントIDEの比較。
- Codex / Claude Code と役割分担できるかの検証。

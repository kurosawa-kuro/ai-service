## ハーネス適性

- ワークフロー組み込み: 高
- 接続方式: CLI、ローカルファイル編集、コマンド実行
- 制約: 実行権限、秘密情報、破壊的操作の扱いを明確にする

## 強み

- コードベースを読んで差分を作れる。
- テスト実行、Makefile、docs 更新まで一連で扱いやすい。
- AGENTS.md による repo 固有ルールとの相性が高い。

## 弱み

- 広い権限を渡すほど事故リスクが上がる。
- 最新仕様や外部サービス情報は確認が必要。
- 大規模変更では task / plan / acceptance criteria がないと散らばりやすい。

## 得意用途

- 小〜中規模の実装。
- 既存コードの修正。
- テスト追加。
- docs と実装の同期。
- CLI / Markdown 台帳の整備。

## 苦手用途

- 明確な仕様がない大規模プロダクト設計。
- 外部サービスの最新料金・規約判断。
- GUI中心の確認が必要な作業。

## 黒澤用途

- ai-service の台帳生成・整備。
- ハーネス設計。
- コードレビュー観点の壁打ち。
- ドキュメントと実装の差分整理。

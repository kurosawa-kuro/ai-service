# 契約管理

黒澤個人の AI サービス契約の台帳。

## 絶対原則

> **フロンティアのハイプランを複数契約しない。**

これは理想形の選択に依存しない、契約判断の不変条件。第一理想・第二理想のどちらを採っていても常に効く。

具体的には、Claude Max と ChatGPT Pro のような**最高峰ティアの同時保持を禁止**する。
高額固定費の二重化そのものが禁止対象であり、「両方使う場面がある」は理由にならない
（片側は標準プランか API 従量で代替する）。

**「フロンティア最安到達プラン」** セクションは恒久管理対象。削除・要約・統合しない。

「各プロバイダのフロンティア最上位モデルに到達できる最安プランはどれか」が、
第一理想と第二理想のどちらを採るかを決める唯一の閾値であり、契約判断の分岐データそのものであるため。
**表に未確認が残っている間は契約変更を実行しない。**

### 番号の意味（重要）

**番号は「移行経路の順序」であって、優先度でも理想度でもない。**

軸が2つあり、両者は一致しないので混同しないこと。

| 軸 | 順序 |
|---|---|
| **移行経路**（＝ファイル番号） | 01 現状 → 02 バーベル → 03 将来方針 |
| **理想としての優先度** | 03 将来方針＝**第一理想** ／ 02 バーベル＝**第二理想** |

第一理想（03）が番号上は最後に来るのは、**そこが到達点だから**であって、優先度が低いからではない。

## 移行経路

```text
絶対原則: フロンティアのハイプランを複数契約しない（全経路で不変）

01 現在の契約 $310/月
   ❌ ハイプラン2枚で原則違反中
      ↓ 片側を降格し、ハイプランを1社に絞る
02 バーベル戦略 ≈$120/月（第二理想）
   ✅ 高額1＋標準1＋API。変化に強い
      ↓ ミドル階層で最上位モデルが十分回るなら
03 将来方針 $60/月（第一理想）
   ✅ コスト最適。ただし安定維持が難しい
```

02 は経路上の中間地点であると同時に、**03 が安定しなかった場合の踏みとどまり先**でもある。
つまり 02 は「通過点」と「退避先」を兼ねる。

- 03（第一理想）はコスト最適だが、**AI の動きが速く、この形を安定させ続けるのが難しい**。
- そのため 03 が維持できないと判断した時点で 02 へ戻す。
- 02 と 03 は競合案ではなく、優先順位のついた目標形。
- どちらに居ても、**絶対原則だけは崩さない**。

01（現在の契約）は、目標2つとは疎結合で管理する（現契約の変更で目標側を書き換えない。逆も同じ）。

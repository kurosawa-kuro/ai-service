# CLAUDE.md

黒澤さん個人の **AI サービス / 技術カタログ**。増え続ける AI サービスを「能力・用途・使い分け・評価」で整理し、次の選択判断に使う個人台帳。手書き Markdown が主。

## 構成

- **`src/`** — カタログ本体（このプロジェクトの実ソース）
  - `src/21_サービス/` サービスごとの台帳（基本情報・強み弱み・黒澤用途・評価ログ）
  - `src/22_用途/` 用途からの逆引き
  - `src/30_評価/` 実タスクの評価ログ
  - `src/23_候補/` 未検証候補
  - `src/10_契約/` 契約管理。**番号＝移行経路順**（`01_現在の契約.md` 現状 → `02_AIフロンティア・バーベル戦略.md` 中間目標=第二理想 → `03_将来方針.md` 最終目標=第一理想）。**番号は優先度ではない**。目標2つと今の契約は疎結合
    - 絶対原則: **フロンティアのハイプランを複数契約しない**（理想形の選択に依存しない不変条件）
    - **削除禁止**: `02_AIフロンティア・バーベル戦略.md` の「フロンティア最安到達プラン」セクションは恒久管理対象。削除・要約・統合しない
  - `src/11_活用戦略/` 契約済みキーの**活用**戦略（`AI-Model API活用戦略.md`）。契約＝`10_契約/`、活用＝`11_活用戦略/` で分ける
  - `src/13_教材連携.md` 外部教材リポ `ai-tool` との連携ルール
  - `src/12_使い分けルール.md` 使い分けの正本 ／ `src/20_モデル/プラン比較.md` プラン比較
- `docs/` — 補助メモ（軽い参照。`docs/tasks/` は作業メモ）
- `env/` — `config.yaml`（非機密）／ `secret.yaml`（ローカル秘密・コミット禁止）

## エントリの足し方

`src/21_サービス/<name>.md` を既存ファイルと同じ見出し構成でコピーして書く。料金 / API / CLI / MCP / ローカル可否は書く時点で確認して確認日を添える。不明な項目は `未確認` のままでよい（断定しない）。

## 方針（軽く・気軽に）

- 個人の Markdown 台帳。DB 化・Web アプリ化・網羅スペック転記を急がない。手書きで気軽に足す。
- **ハーネスは AI（モデル）を導くためのもので、owner を縛るためのものではない。** 承認ゲート・検証レベル・タスク契約・scope-creep 検出のような重いプロセスは持たない（以前それで owner 自身の使い勝手を殺した反省。AI ではなくユーザーをハーネスしてしまうのは失敗）。
- 秘密情報（API キー・接続文字列・cookie）はコミットしない。自分用のメモ（契約プラン等）は `src/` に自由に書いてよい。

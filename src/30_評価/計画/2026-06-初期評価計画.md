# 2026-06 初期評価計画

これは実評価ログではなく、最初にどの順番で検証するかの計画。

## 目的

台帳化したサービス / 技術に、実タスクベースの評価ログをつける。

## 評価バッチ 1: コーディング / ハーネス

| 対象 | タスク | 見ること |
|---|---|---|
| Codex / Codex CLI | 小〜中規模の実装 + テスト + docs更新 | 差分品質、検証、作業継続性 |
| Claude Code | 設計レビュー + 実装方針作成 | 仕様矛盾、構造化、レビュー品質 |
| GitHub Copilot | IDE補完 | 補完の速さ、邪魔しなさ、精度 |
| Cursor | 複数ファイル編集 | IDE体験、差分管理、CLI勢との競合 |
| Ollama | ローカル要約 / 分類 | 速度、品質、秘匿用途の実用性 |

## 評価バッチ 2: 調査 / 資料

| 対象 | タスク | 見ること |

| Perplexity | AIサービスの最新仕様調査 | 出典品質、一次情報への到達性 |
| NotebookLM | PDF / 講座資料の要約 | 手元資料理解、引用、抜け漏れ |
| Gamma | 講座スライド初稿 | 日本語品質、修正容易性、構成力 |

## 評価バッチ 3: 自作ルーター / 運用

| 対象 | タスク | 見ること |
